Why not Electron?

Electron ships an entire Chromium browser and a Node.js runtime inside every app. That means a simple file-scanning utility would carry ~150â€“300 MB of overhead, spawn multiple background processes, and idle at 100â€“200 MB of RAM before the user even clicks a button. dragoNotebook targets developers whose machines are already running IDEs, browsers, Docker containers, and language servers â€” every megabyte of headroom matters.

Tauri sidesteps this entirely. Instead of bundling a browser, it uses the WebView already installed on the operating system â€” WebView2 on Windows (Chromium-based, pre-installed on Windows 10 21H2+ and all Windows 11 machines), WebKit on macOS and Linux. The result: a ~4â€“8 MB installer that idles at ~30â€“50 MB of RAM with zero Node.js processes running in the background.

Why not a pure-Rust GUI (egui, iced, etc.)?

Pure-Rust immediate-mode frameworks like egui (eframe) compile to a single binary with zero external dependencies â€” genuinely impressive for certain workloads. But dragoNotebook's UI is a form-driven configuration tool: file trees, checkboxes, dropdown menus, text inputs, collapsible panels, tier-gated feature visibility, and responsive layout. This is exactly the kind of interface that HTML/CSS was built to express in a few lines and that immediate-mode Rust GUIs require hundreds of lines of manual layout code to replicate.

The Node.js question

A common concern with web-based desktop frameworks: "does it require Node.js?" With Tauri, the answer is no â€” not at runtime. dragoNotebook's frontend is static HTML, CSS, and vanilla JavaScript. There is no npm install, no node_modules, no webpack, no dev server. The build pipeline is pure cargo. Users who download the app get a native installer; Node.js never enters the picture.

This matters for the same reason the Electron comparison matters: developer workstations are resource-constrained. A desktop app that silently spins up a Node.js event loop, an HTTP server, and a file watcher just to render a config form is paying an engineering tax that dragoNotebook avoids entirely.

Where egui shines instead

Pure-Rust frameworks are the better choice when the application is the rendering â€” real-time data visualization, interactive charts at 60 fps, GPU-accelerated canvases where every frame repaints from computed data with zero serialization overhead. For those workloads, immediate-mode Rust drawing directly on an OpenGL surface has a clear performance advantage over shipping data across an IPC bridge to a browser renderer.

Bottom line: Tauri gives dragoNotebook a polished, accessible, CSS-styled desktop experience with built-in installers and auto-update â€” at a fraction of Electron's resource cost and without requiring Node.js at build or runtime. The trade-off (OS WebView dependency, slightly slower cold start) is well worth it for a form-driven developer tool that prioritizes low overhead and cross-platform reach.
